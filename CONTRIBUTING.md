# Contribuer à YeliSport

Créez une branche courte depuis `develop`, ajoutez les tests pertinents et exécutez
`make lint test` avant d'ouvrir une pull request. Une pull request doit décrire le
problème, la solution, les migrations éventuelles et la stratégie de retour arrière.

Ne commitez jamais `.env`, une clé, un jeton ou des données personnelles.
